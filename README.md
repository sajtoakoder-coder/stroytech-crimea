# СтройТех — Крым

Лендинг строительной компании «СтройТех» (Крым): строительство и проектирование
коттеджей, аренда спецтехники. Плюс отдельная страница политики
конфиденциальности. Чистая статика — без сборки, фреймворков и зависимостей.

## Структура проекта

```
/
├── index.html            # главная страница (лендинг)
├── privacy.html          # политика конфиденциальности
├── assets/
│   ├── css/
│   │   └── styles.css     # общие стили обеих страниц
│   ├── js/
│   │   └── main.js        # вся логика главной: меню, тема, слайдер, форма…
│   ├── img/               # фотографии, логотип, QR-коды
│   └── icons/
│       └── favicon.svg    # иконка сайта
├── .gitignore
└── README.md
```

Несколько технических деталей:

- Маленький анти-FOUC скрипт переключения темы оставлен инлайном в `<head>`
  каждой страницы — он обязан отработать до первой отрисовки, иначе тема мигает.
- Стили страницы политики заскоуплены в `styles.css` под селектор
  `.page-privacy` (класс на её `<body>`), чтобы не влиять на главную.
- Декоративные иконки — инлайновые `<svg>` в разметке: они перекрашиваются под
  тему через `currentColor`.
- Теги `og:image` / `twitter:image` и JSON-LD ссылаются на изображение по
  абсолютному URL продакшен-домена — этого требуют соцсети и поисковые роботы.

## Локальный запуск

Нужен любой статический HTTP-сервер (открывать через `file://` нежелательно —
не будут работать относительные пути и `fetch`).

```bash
# Python 3 (есть на macOS «из коробки»)
python3 -m http.server 8000
```

Затем открыть <http://localhost:8000/>. Страница политики —
<http://localhost:8000/privacy.html>.

Альтернатива — расширение **Live Server** в VS Code: правый клик по
`index.html` → «Open with Live Server».

## Деплой на Vercel

Проект статический, шаг сборки не нужен.

**Вариант 1 — через сайт (drag-and-drop):**

1. Зайти на [vercel.com](https://vercel.com), «Add New… → Project».
2. Импортировать репозиторий из Git **или** просто перетащить папку проекта.
3. Framework Preset: **Other**. Build Command — пусто. Output Directory —
   корень проекта (`.`). Нажать **Deploy**.

**Вариант 2 — через CLI:**

```bash
npm i -g vercel   # один раз
vercel            # превью-деплой
vercel --prod     # продакшен
```

На вопросы про build/output можно отвечать пустыми значениями — Vercel
раздаёт файлы как есть.

> Примечание: форма отправляет данные на `POST /api/lead`. На чистой статике
> этот эндпоинт вернёт 404 и форма покажет сообщение об ошибке. Чтобы заявки
> работали, добавьте serverless-функцию в `api/lead.js` (Vercel Functions) или
> укажите свой обработчик в `assets/js/main.js`.
